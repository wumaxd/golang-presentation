Pros/Cons of Go

Pros:

  • Fast builds
  • Performance
  • Comprehensibility: easy & quick to develop, focus on readability
  • Linguistic stability: The "Go 1 promise"
  • Memory safety
  • Static linking
  • great standard library and huge ecosystem of open source libraries

Cons:

  • sometimes too simple & reduced: Slice operations
  • missing generics lead to multiple implementations of the same algorithm
2

Recommended open source libraries

  • github.com/gorilla/mux: HTTP-request multiplexer, compatible with standard HTTP-Server
    • Gorilla Toolkit: many useful tools around HTTP-Server
  • github.com/google/uuid: UUID data structure
  • github.com/spf13/cobra: CLI library (Docker & K8s is using it)
  • github.com/sirupsen/logrus: Often used logging library
  • gocloud.dev: cloud development kit for cross-cloud platform interaction (â†’ multi- & hybrid-cloud)
